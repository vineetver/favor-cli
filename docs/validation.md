## Validation method

Ground truth values are generated by running the R STAAR package on controlled inputs. The R script (`test_reference/generate_r_ground_truth.R`) produces a JSON file with exact p-values, weights, residuals, and kernel matrices. Rust `#[cfg(test)]` code loads this JSON and asserts our output matches R's within tolerance.

```
R STAAR 0.9.8 (reference implementation)
        |
        v
generate_r_ground_truth.R --> ground_truth.json
                                    |
                                    v
                              Rust #[cfg(test)]
                              loads JSON, runs our code,
                              asserts |actual - expected| < tolerance
```

If R says `Burden(1,25) = 0.4155`, our Rust must produce the same value.

## Internal invariant tests

Beyond R validation, the Rust implementation enforces structural and mathematical invariants:

### Sparse-dense parity
The carrier-indexed sparse scoring path (O(total_MAC)) must produce identical U vectors and K matrices as the naive dense path (O(n_samples x n_variants)). Tested for both continuous and binary traits.

| Property | Tolerance | Test |
|----------|-----------|------|
| U: sparse == dense (continuous) | 1e-10 | `sparse_dense_parity_continuous` |
| U: sparse == dense (binary) | 1e-10 | `sparse_dense_parity_binary` |
| K: sparse == dense (continuous) | 1e-10 | `sparse_dense_parity_continuous` |
| K: sparse == dense (binary) | 1e-10 | `sparse_dense_parity_binary` |
| Burden/SKAT/ACAT-V/STAAR-O: raw G == U/K oracle | 1e-9, SKAT 1e-6 | `raw_path_matches_oracle` |
| Burden/SKAT/ACAT-V/STAAR-O: sparse == U/K oracle | 1e-9, SKAT 1e-6 | `sparse_path_matches_oracle` |

The U/K oracle entry point is `score::run_staar_from_sumstats`, which is directly compared to R STAAR's `STAAR()` by `staar_continuous_matches_r`. The two parity tests above extend that anchor to the raw and sparse scoring paths, covering every Burden/SKAT/ACAT-V variant including the STAAR-O omnibus.

### Algebraic properties
These catch index alignment bugs, hidden reorderings, and accumulation errors.

| Property | What it verifies | Test |
|----------|-----------------|------|
| Permutation invariance | Reordering carriers produces consistent U/K under the same permutation | `permutation_invariance` |
| Duplicate idempotence | [v, v, v] produces identical U values and uniform K | `duplicate_variant_idempotence` |
| Full mask == no mask | Slicing U/K by all indices == unsliced | `full_mask_equals_unsliced` |
| Subset slice == recompute | slice(U,K,[1,3,5]) == recompute from carrier subset | `random_subset_slice_equals_recompute` |
| Dense carrier accumulation | 50% carrier rate, tests f64 accumulation accuracy | `dense_carrier_variant` |
| Singleton gene | MAC=1 variant, U == residual at carrier sample | `singleton_gene` |
| Empty gene | Zero variants, returns 0x0 matrices | `empty_gene` |

### Storage invariants
The genotype store (`sparse_g.bin` + `variants.parquet` + `membership.parquet`) is validated by `store_validate::validate_store()`:

| Invariant | What it catches |
|-----------|-----------------|
| variant_vcf == row index | Misaligned metadata |
| Offsets monotonic and in-bounds | Corrupted files, misaligned writes, silent truncation |
| sample_id < n_samples, dosage in {1,2} | Out-of-range carrier entries |
| No duplicate sample_ids per variant | Hidden duplicate carriers |
| sample_ids strictly increasing per variant | Unsorted carriers (breaks merge-join) |
| membership variant_vcf < n_variants | Invalid gene assignment references |
| Variants sorted by (pos, ref, alt) | Broken variant_vcf ordering |
| Membership sorted by variant_vcf per gene | Broken gene iteration |
| sparse_g.n_variants == VariantIndex.len() | Storage/metadata count mismatch |
| vid from metadata == stored vid | Cross-layer misalignment |
| maf > 0 implies non-zero carriers | Storage/frequency inconsistency |

### Alignment tests
These verify that masks, weights, and metadata are correctly aligned to variant_vcf:

| Test | What it verifies |
|------|-----------------|
| Gene variant_vcfs correct | membership.parquet matches expected gene assignment |
| Overlapping genes share variant | variant in two genes is reachable from both |
| vid roundtrip | vid string <-> variant_vcf is bijective |
| Weight channel alignment | weight_channels[ch][v] == entries[v].weights[ch] for all ch, v |
| compile_mask filters correctly | MAF + predicate produce correct local indices |
| Mask composition | (A OR B) == union(A, B) |
| Region == mask | WHERE pos BETWEEN X AND Y == scan over entries[] |

## Known method differences vs R

These are intentional implementation choices, not bugs:

### SKAT p-value approximation
- **R**: Eigenvalue saddlepoint (primary), Liu 2-moment (fallback)
- **Rust**: Liu 4-moment matching (always)
- **Impact**: < 0.2% relative difference. Both are valid approximations for Q ~ sum(lambda_j * chi^2_1). The 4-moment method is more accurate than R's 2-moment fallback.

### ACAT-V per-variant test (continuous)
- **R**: t-test with (n-k-1) degrees of freedom
- **Rust**: Chi-squared(1) test
- **Impact**: Asymptotically equivalent for large n. The t-test is slightly more conservative for small samples.

### SPA root-finding
- **R**: Newton-Raphson with bisection fallback
- **Rust**: Bisection only
- **Impact**: Same statistical result when both converge. Bisection is slower but unconditionally stable.

## Not yet validated against R

| Component | What is tested | What is not | Blocker |
|-----------|---------------|-------------|---------|
| MetaSTAAR cross-study merge | U/K scaling vs R (`meta_staar_validation`) | Cross-study covariance merge in `meta.rs` | Multi-study test setup |
| SCANG threshold search | Window construction (`masks::tests::scang_*`); R parameter loading | Threshold search vs R | Monte Carlo simulation needed |
| AI-STAAR | Two-population unit test (`ancestry::tests::ai_staar_two_populations`) | End-to-end vs R | Ancestry-stratified reference data |
| MultiSTAAR joint test | Per-trait STAAR-O matches R; Rust unit tests for combine | Joint omnibus vs R (`multi_staar_o` is NULL in current reference) | Multi-trait reference where R returns non-NULL |
| SPA (binary traits) | CGF/derivative correctness (`stats::tests::cgf_*`, `k1_at_zero_is_zero`, `k2_at_zero_equals_variance`); pipeline plumbed via `burden_spa`, `acat_v_spa` | Per-variant SPA p-value vs R STAAR_Binary_SPA | Balanced binary reference where R returns non-NULL |

## Reproducing validation

```bash
# Generate ground truth from R (requires R + STAAR installed)
srun -p xlin -c 4 --mem=8G -t 0-00:10 bash -c \
  'module load R/4.4.3-fasrc01 cmake && Rscript test_reference/generate_r_ground_truth.R'

# Run Rust tests against ground truth
srun -p xlin -c 8 --mem=16G -t 0-00:10 cargo test ground_truth

# Run all tests (130 tests including invariant checks)
srun -p xlin -c 8 --mem=16G -t 0-00:10 cargo test
```
